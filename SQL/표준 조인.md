## INNER JOIN
**교집합**의 개념으로 기준이 되는 키에 따른 컬럼값이 존재하는 것만 병합해서 결과를 만든다.

## OUTER JOIN
**합집합**의 개념으로 기준이 되는 키에 따른 컬럼값이 존재하지 않더라도 모든 튜플을 병합하는 것을 말한다.\
```Outer Join```은 ```Left Outer join```과 ```Right Outer Join```, ```Full Outer Join```으로 나눌 수 있다.

## LEFT OUTER JOIN
왼쪽 테이블의 모든 행을 포함시키면서 Join을 수행한다.
```sql
SELECT
A1.ACTOR_NO ACT_NO1,
A2.ACTOR_NO AC_NO2,
A2.MOVIE_NO,
A1.NAME,
A1.GENDER
FROM ACTOR A1 LEFT OUTER JOIN APPR A2 ON A1.ACTOR_NO = A2.ACTOR_NO;
```
+ ACTOR 테이블과 APPR 테이블을 LEFT OUTER JOIN 하며,
+ ACTOR의 ACTOR_NO와 APPR의 ACTOR_NO를 기준키로 비교하여 같은 값을 갖는 행을 포함한 ACTOR의 모든 행을 출력한다.

## RIGHT OUTER JOIN
오른쪽 테이블의 모든 행을 포함시키면서 조인을 수행한다.
```sql
SELECT
A1.ACTOR_NO ACT_NO1,
A2.ACTOR_NO AC_NO2,
A2.MOVIE_NO,
A1.NAME,
A1.GENDER
FROM ACTOR A1 RIGHT OUTER JOIN APPR A2 ON A1.ACTOR_NO = A2.ACTOR_NO;
```
+ ACTOR 테이블과 APPR 테이블을 RIGHT OUTER JOIN 하며,
+ ACTOR의 ACTOR_NO와 APPR의 ACTOR_NO를 기준키로 비교하여 같은 값을 갖는 행을 포함한 APPR의 모든 행을 출력한다.

## FULL OUTER JOIN
왼쪽 테이블의 모든 행과, 오른쪽 테이블의 모든 행을 포함하는 조인을 수행한다.
```sql
SELECT
A1.ACTOR_NO ACT_NO1,
A2.ACTOR_NO AC_NO2,
A2.MOVIE_NO,
A1.NAME,
A1.GENDER
FROM ACTOR A1 FULL OUTER JOIN APPR A2 ON A1.ACTOR_NO = A2.ACTOR_NO;
```
+ ACTOR 테이블과 APPR 테이블을 RIGHT OUTER JOIN 하며,
+ ACTOR의 ACTOR_NO와 APPR의 ACTOR_NO를 기준키로 비교하여 같은 값을 갖는 행을 포함한 ACTOR와 APPR의 모든 행을 출력한다.
## NATURAL JOIN
조인의 대상이 되는 두 테이블에서 같은 이름의 컬럼에 대해서는 **동일한 컬럼값을 가지는 행만 병합되는 조인**
```sql
SELECT * FROM ACTOR NATURAL JOIN APPR;
```
+ 먼저 ACTOR_NO의 값이 동일한 행은 1235, 3398이다.
+ 여기에 GUARANTEE까지 같은 값을 가진 행은 1235뿐이다.

※ 주의 : SQL Server에서는 NATURAL JOIN을 지원하지 않는다. NATURAL JOIN은 조인 조건을 내포하고 있으므로 **ON절을 사용할 수 없으며** SELECT절의 컬럼명에도 별칭을 함께 표시할 수 없다.

## CROSS JOIN
왼쪽 테이블의 각 행에 대한 오른쪽 테이블 모든 행의 대응을 조합하여 결과를 출력하는 조인이다.\
왼쪽 테이블이 M행이고 오른쪽 테이블이 N행이면 CROSS JOIN의 결과는 ```M × N```행이 된다.
```sql
SELECT S1.NAME, C1.NAME
FROM STUDENT S1, CLUB C1;
```
- 포준 조인이 아닌 경우 WHERE절 없이 FROM절에 테이블들을 나열하면 CROSS JOIN이 된다.
