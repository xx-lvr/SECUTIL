## 네트워크 주소, 브로드캐스트 주소
**IP 주소**에는 호스트에 할당할 수 없는 두 가지 특수한 주소가 존재한다.
바로 **네트워크 주소와 브로드캐스트 주소**이다.
![alt text](<../image/network, broadcast.png>)
+ **네트워크 주소**: 해당 네트워크 자체를 식별하기 위한 주소
+ **브로드캐스트 주소**: 네트워크에 속한 모든 호스트에게 데이터를 전송하기 위한 주소

이 두 주소는 모든 IP 클래스와 서브넷에서 공통적으로 적용되며,
PC나 서버에 할당할 수 없는 주소이므로 반드시 구분해서 이해해야 한다.

한 네트워크를 대표하거나 전체 호스트에게 데이터를 전달하기 위한 목적의 주소를
일반 호스트에 할당하면 네트워크 오류가 발생할 수 있으므로
사용 불가능한 주소로 분류된다.

## 예시
IP 범위: ```192.168.10.0 ~ 192.168.10.255```

이 범위에서

+ ```192.168.10.0``` → 네트워크 주소
+ ```192.168.10```이라는 네트워크 자체를 의미
+ ```192.168.10.255``` → 브로드캐스트 주소

해당 네트워크에 속한 모든 호스트에게 데이터 전송 시 사용

```192.168.10.1 ~ 192.168.10.254``` → 사용 가능한 호스트 주소

## 정리
### 네트워크 주소 (Network Address)
+ 호스트 ID가 모두 0인 주소
+ 네트워크 자체를 식별하기 위한 주소
+ 호스트에 할당 X

### 브로드캐스트 주소 (Broadcast Address)
+ 호스트 ID가 모두 1인 주소
+ 네트워크 내 모든 호스트에게 데이터를 전송하기 위한 주소
+ 호스트에 할당 X
