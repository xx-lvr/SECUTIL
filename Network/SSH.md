## SSH란?

**SSH(Secure Shell)** 는 원격 호스트에 안전하게 접속하기 위한 보안 프로토콜이다.\
네트워크 상에서 데이터를 암호화하여 전송하므로, 도청·위변조 위험을 줄일 수 있다.

---

## 왜 쓰는가?

기존의 **Telnet**은 데이터를 평문(암호화 없이) 전송하기 때문에
아이디, 비밀번호가 그대로 노출되는 치명적인 보안 문제가 있었다.
이를 해결하기 위해 **암호화 통신을 지원하는 SSH**가 등장.

---

## 동작 원리

SSH의 핵심은 **Key(키)** 이다.\
클라이언트와 서버는 키를 이용해 서로를 인증하고,
인증이 완료되면 암호화된 통신 채널을 생성하여 데이터를 주고받는다.

SSH에서는 **비대칭키 방식과 대칭키 방식을 함께 사용**한다.

---

## 비대칭키 (공개키 암호 방식)

- **공개키(Public Key)** 와 **개인키(Private Key)** 한 쌍으로 구성
- 공개키: 서버에 저장
- 개인키: 사용자(클라이언트)가 보관

### 동작 과정
1. 클라이언트가 서버에 접속 요청
2. 서버는 공개키로 암호화된 데이터를 전송
3. 클라이언트는 자신의 개인키로 복호화
4. 이를 통해 서버는 사용자를 인증함

-> **주 용도: 사용자 인증**

---

## 대칭키 암호 방식

- 하나의 동일한 키로 암호화와 복호화를 수행
- 처리 속도가 빠름

### SSH에서의 역할
- 비대칭키 인증이 끝난 후
- 서버와 클라이언트가 세션용 대칭키를 생성
- 이후 데이터 통신은 대칭키로 암호화하여 진행

-> **주 용도: 실제 데이터 전송**

---

## 정리

- **비대칭키**: 신원 확인(인증)
- **대칭키**: 빠르고 안전한 데이터 통신
- SSH는 두 방식을 결합하여 **보안성과 성능을 모두 확보**
